# Media prompts for spine v1

These prompts are designed for launch assets, README visuals, and a short product GIF.

## Prompt 1: README hero image

```text
Use case: productivity-visual
Asset type: README hero image
Primary request: Create a polished launch visual for a developer tool named "spine" that turns an unfamiliar codebase into a verified onboarding guide.
Scene/backdrop: Editorial product illustration on a warm off-white background with subtle paper texture, faint graph lines, and soft code-window overlays.
Subject: A laptop showing a small Mermaid architecture map on the left and a clean onboarding document on the right, with a highlighted reading order and the phrase "verified edges only".
Composition: Wide horizontal composition for a GitHub README hero, centered product screen, generous whitespace, crisp framing.
Style: Premium technical editorial illustration, not cartoonish, not generic SaaS purple. Use ink blue, graphite, muted teal, and warm amber accents.
Text to include verbatim: spine, verified onboarding for codebases
Important details: Make the architecture map feel trustworthy and compact, not chaotic. Show a sense of clarity replacing confusion. Keep typography sharp and legible.
Avoid: purple gradients, cyberpunk aesthetics, glowing neon, cluttered dashboards, fake code walls, stock-photo people, excessive 3D glassmorphism
Output: high-resolution banner suitable for README and blog header
```

## Prompt 2: Social/blog cover image

```text
Use case: ads-marketing
Asset type: blog and social cover image
Primary request: Create a launch cover image for a tool called "spine" with the message "Find the real entry point."
Scene/backdrop: Minimal editorial layout with layered repository cards, one messy and one organized into a small verified architecture map.
Subject: A visual transformation from noisy file tree to concise diagram and reading list.
Composition: Strong left-to-right before/after flow, bold title area, enough safe space for blog cropping.
Style: Modern product storytelling, restrained, high-contrast, thoughtful, designed for engineers.
Text to include verbatim: spine, Find the real entry point.
Color direction: warm neutral background, dark ink text, teal and amber as the only accent colors.
Avoid: exaggerated futuristic UI, purple branding, random floating icons, overly abstract shapes
Output: landscape cover image for blog post and social preview
```

## Prompt 3: GIF storyboard prompt

```text
Use case: ui-mockup
Asset type: 8-second launch GIF storyboard
Primary request: Create a sequence of frames for a product GIF showing how spine turns an unfamiliar repository into a verified onboarding guide.
Scene/backdrop: Clean terminal-first product demo with subtle paper-texture background and calm editorial framing.
Storyboard:
Frame 1: A repository tree and README feel noisy and overwhelming.
Frame 2: Terminal command appears: onboard . --map-only
Frame 3: A compact Mermaid architecture map appears with the label "verified edges only".
Frame 4: Terminal command appears: onboard .
Frame 5: ONBOARDING.md opens with TL;DR, reading order, subsystems, and gotchas.
Frame 6: Final frame shows the message "Start in the right place."
Style: Crisp terminal UI, understated motion, professional product launch aesthetic, no gimmicks.
Color direction: ink, warm white, teal, amber.
Important details: Prioritize legibility and simple transitions over flashy effects. Make the result feel calm, credible, and useful.
Avoid: glitch effects, neon, exaggerated motion blur, fake typing overload, tiny illegible text
Output: storyboard frames for a short looping GIF
```
